11. Blok perintah yang digunakan sampai di interupsikan ​


Untuk menjalankan blok perintah tanpa diinterupsi, Anda dapat menggunakan loop atau struktur pengulangan yang cocok dalam bahasa pemrograman yang Anda gunakan. Berikut adalah contoh blok perintah yang menggunakan loop while untuk menjalankan perintah sampai diinterupsi:

contoh jika python

while True:

   # Blok perintah yang ingin dijalankan

   # ...

   

   # Mengecek apakah ada interupsi

   interupsi = input("Apakah ingin menginterupsi? (ya/tidak): ")

   

   if interupsi.lower() == "ya":

       break  # Keluar dari loop jika ada interupsi

Dalam contoh di atas, blok perintah yang ingin dijalankan ditempatkan di dalam loop while True yang akan terus berjalan selama kondisinya benar (di sini kondisinya selalu True). Di setiap iterasi loop, program akan memeriksa apakah ada interupsi yang dimasukkan oleh pengguna. Jika pengguna memasukkan "ya", maka loop akan diinterupsi dengan menggunakan pernyataan break.

Anda dapat menyesuaikan blok perintah yang ingin dijalankan sesuai kebutuhan Anda di dalam loop tersebut. Pastikan untuk menyesuaikan dengan bahasa pemrograman yang Anda gunakan dan menambahkan interupsi yang sesuai dengan cara yang cocok untuk bahasa tersebut.

18. Jelaskan proses interupsi, contohkan 2 kejadiannya


Jawaban:

Komponen yang mengalami Abnormalitas akan mengirimkan sinyal interrupt_request pada processor yang nantinya akan ditangani oleh intterupt_handler

Intterupt Handler :

Banyak sekali hal yang dapat menimbulkan Interupsi pada sistem. Interrupt Handler biasanya akan membagikan interupsi ke 3 tingkatan error level.

--> Warning : OS akan mencoba membuat logs tentang interupsi tersebut

---> Critical : OS akan langung men-Shutdown System

---> Low-Level : OS akan mencoba meng-Ignore begitu saja

                        (Cek jawaban lengkapnya di thread lain)

Intterupt Cause :

 Interupsi biasanya disebabkan perangkat lunak mendeteksi kesalahan pada komponen-komponen didalamnya. Jika masalah tersebut muncul dari eksternal maupun komponen yang tidak terpakai, maka software akan mengirimkan sinyal abnormalitas yang nantinya akan memproduksi warning ataupun low-level error. Jika hal tersebut mengenai komponen vital, software tersebut akan mengirimkan Critical Error dan menshutdown tree-process yang sedang berjalan

Interrupt Process :  

 Pertama-tama, Perangkat Lunak akan mengirimkan sinyal interrupt_req pada processor yang nantinya, processor akan mencoba me-Resolve masalah tersebut dengan memunculkan interrupt_handler yang berjalan di dalam background, intterupt handler akan tertrigger melalui service. Pada OS, jika salah satu sub-komponen mengalami interupsi maka Intterupt handler akan me-Launch Troubleshooter yang akan mendeteksi tahap 2 abnormalitas. Dengan Troubleshooter, biasanya OS akan mencoba memperbaiki sendiri dengan cara-cara yang telah di set oleh developer OS.

31. Jelaskan bagaimana OS bekerja dalam menangani Interupsi


Jawaban:

Salah satu karakteristik komputer (PC) yang sering kali ditunjukkan adalah ia selalu mengerjakan perintah berurut, langkah demi langkah.

Penjelasan:

Prosesor dapat saja menerima sebuah sinyal penting dimana ia harus segera menanganinya dan menunda terlebih dahulu apa yang sedang dikerjakannya saat ini. Sinyal penting tersebut disebut sebagai interrupt. Prosesor dapat menerima dua jenis interrupt, yaitu hardware interrupt dan software interrupt. Sesuai namanya, hardware interrupt adalah interrupt yang dikirim oleh hardware melalui jalur khusus. Contoh hardware interrupt adalah sinyal yang dikirim ke prosesor saat tombol keyboard ditekan.

Jawaban:

Menyaring Interupsi sesuai dengan tingakatan error level.

Penjelasan:

OS tanpa disadari sering mengalami interupsi di bagian background. OS menangani interupsi sesuai dengan error level yang dibagi menjadi :

- Critical Error

- Warning

- Low Level Error

  1. Critical Error :

 Critical error adalah tingkatan yang oleh OS biasanya akan diperhatikan, jika ada suatu program mengalami tingkatan Critical Error maka OS akan menimbulkan layar Critical Error dan kemudia merestart dengan me-launching troubleshoot. Pada Windows biasanya akan memunculkan Blue-Screen sebagai pertanda Critical Error.

        Penyebab:

  Critical Error biasanya disebabkan oleh abnormal file-system atau interupsi pada OS library, pada Windows.. Interupsi pada nt.dll ataupun base.dll akan langsung membuat Komputer anda jentet.

  2. Warning

 Pada tingkatan Warning, biasanya OS hanya akan mencoba untuk menyaring error level tersebut kedalam logs, pada Windows XP.. tingkatan Warning akan memunculkan pop-up warning, cara tersebut kemudian banyak disalah-gunakan untuk menciptakan virus yang memunculkan error level warning. Pada Windows Logs, Tanda warning memiliki warna kuning.

   3. Low-Level Error

  Error ini adalah error yang sudah lumrah terjadi setiap menitnya di OS background, OS biasanya akan hanya mencoba meng-Ignore error tersebut.
