5. Jelaskan macam-macam dns?​


DNS atau Domain Name Resolver merupakan suatu fungsi dimana alamat domain ditranslasi / diubah menjadi alamat IP. Berbagai macam DNS terbagi menjadi:

1. DNS Recursive

   DNS Recursive bertindak sebagai orang yang berada ditengah-tengah dalam menjalankan dns query. Ketika client meminta dns query, dns recursive akan menyediakan hasil berupa cache untuk mempercepat dns resolve. Misal Cloudflare 1.1.1.1

2.DNS Root Nameserver

  DNS Root Nameserver merupakan fitur dimana resolving tsb dimulai dari server ISP yang kemudian akan diteruskan kepada TLD DNS. TLD Nameserver akan merespons alamat IP yang akan dituju dan kembali lagi ke client.

3.TLD Nameserver

  Berbeda dengan DNS Root Nameserver, TLD Nameserver memiliki server tersendiri untuk setiap TLD yang dimilikinya. misal untuk akses dari luar indonesia harus melalui DNS Root Nameserver, sedangkan untuk visitor domestik bisa melalui server TLD Di indonesia misal PANDI.

19. apa yang dimaksud dengan dns dan apa perbedaan public dns dengan dns default


DNS yaitu Sebuah sistem yang menyimpan informasi tentang nama host ataupun nama domain dalam bentuk basis data tersebar (distributed database) di dalam jaringan komputer.

Jawaban:

DNS atau Domain Name System berfungsi sebagai penerjemah/pengubah dari alamat situs ke bentuk IP address.

Simplenya DNS adalah buku telepon agar perangkat kita dapat menemukan IP address dari suatu situs web dengan lebih cepat.

Contoh DNS

Beberapa DNS publik yang sudah sangat umum yaitu DNS Google (8.8.8.8) dan OpenDNS (208.67.222.222). Banyak penyedia Public DNS lainnya yang memberikan layanan DNS secara gratis seperti Cloudflare, AdGuard, Quad9, Verisign, dll. yang menyediakan keunggulannya masing-masing.

Semua tergantung User dalam memilih ingin menggunakan DNS yang mana.

Perbedaan

DNS Publik atau Public DNS memiliki beberapa keunggulan tersendiri tergantung dari penyedia DNS-nya. Terkadang menggunakan Public DNS jauh lebih menguntungkan dalam beberapa hal dibandingkan menggunakan DNS default bawaan provider.

Contoh Penggunaan

Menggunakan Public DNS seperti Google DNS lebih menguntungkan daripada menggunakan DNS bawaan provider karena server Google sendiri sudah sangat canggih, jauh lebih maju dibanding server yang digunakan provider internet tertentu sehingga dalam beberapa kasus jika kita menggunakan Google DNS atau DNS publik lainnya maka aktivitas browsing kita bisa jauh lebih lancar dan cepat.

Lalu biasanya DNS bawaan dari provider internet kita sudah dimodifikasi agar menampilkan iklan yang mempromosikan produk & layanan mereka. Beberapa pengguna pastinya merasa terganggu dengan iklan-iklan ini. Hal ini tentunya tidak ada jika menggunakan Public DNS dari Google, OpenDNS, Cloudflare, dsb.

Catatan

Yang perlu diperhatikan adalah pastikan untuk selalu menggunakan Public DNS dari penyedia yang memang terpercaya seperti Google, Cloudflare, OpenDNS, dan sejenisnya. Hal ini karena aktivitas browsing dapat dicatat oleh server DNS. Agar tidak disalah-gunakan maka harus menggunakan penyedia Public DNS yang terpercaya.

1. Membuka File

Langkah pertama untuk melakukan setting DNS ubuntu server yaitu dengan membuka file yang diperlukan. Perlu diketahui bahwa file yang diperlukan ini masih ada, namun mungkin bentuknya akan menjadi symlink. Yang diketahui dilakukan pengendalian oleh layanan lainnya.

Untuk itulah tak diperbolehkan untuk dilakukan pengeditan dengan cara manual. Istilah yang namanya systemd-resolved merupakan suatu layanan yang diketahui memberikan resolusi DNS Name. Biasanya diperuntukan bagi aplikasi lokal maupun layanan yang mampu dikonfigurasi memakai Netplan.

2. Menyimpan di Directory

Tahap berikutnya dalam melakukan pengaturan pada DNS Ubuntu server yakni melakukan penyimpanan atas file konfigurasi dari Netplan. Kemungkinan pengguna nantinya akan menjumpai satu ataupun dua jenis file YAML pada bagian directory ini.

Kemudian bisa saja terjadi perbedaan atas nama file dari satu pengaturan dengan pengaturan lainnya. Umumnya file ini mempunyai nama tersendiri yang kemungkinan pada sistem yang dimiliki oleh pengguna akan berbeda.

Dengan file ini pengguna berkemungkinan untuk melakukan konfigurasi interface jaringan. Hal ini juga mencakup tentang gateway kemudian alamat IP, lalu DNS nameserver dan lainnya. Nantinya untuk melakukan pembukaan terhadap konfigurasi ini bisa menggunakan teks editor.

3. Mengubah IP Menjadi DNS Server Pilihan

Perlu diketahui bahwa untuk melakukan konfigurasi DNS ini pengguna perlu melakukan perubahan terhadap alamat IP sekarang ini. Hingga alamat IP tersebut menjadi DNS server yang menjadi pilihan. Contohnya ketika pengguna ingin mengubah DNS server cloudflare.

Jika demikian maka pengguna perlu melakukan perubahan pada bagian baris address yang ada. Dalam penulisan yang dipakai, DNS server perlu dipisah dengan koma. Nantinya dalam hal ini pengguna dapat melakukan penambahan dua name server lebih.

4. Mengedit File

Ketika nantinya tidak didapati entri, maka dapat dilakukan penambahan pada bagian bawah dari blok nama interface. Di waktu melakukan pengeditan file YAML, pengguna perlu memastikan bahwasanya sudah memenuhi standar indentasi dari kode YAML itu sendiri.

Apabila kemudian didapati kesalahan dalam melakukan konfigurasi, maka Netplan tak akan melakukan parsing file. Jika sudah selesai tinggal simpan saja file kemudian terapkan perubahan yang dilakukan.

5. Menghasilkan Konfigurasi

Jika sejumlah langkah sebelumnya sudah dijalankan dengan baik. Nantinya Netplan akan menghadirkan yang namanya file konfigurasi bagi layanan yang dilakukan penyelesaian dengan memakai systemd.

Guna melakukan tahapan verifikasi terhadap DNS resolver bahwasanya sudah dalam keadaan diatur, maka perlu untuk menjalankan sejumlah perintah tertentu.

Dimana perintah tersebut berkemungkinan untuk mencetak banyak informasi. Ketika nantinya pengguna sudah memakai filter string, maka selanjutnya bagian outputnya akan tampak terlihat. Untuk itulah pengguna dituntut untuk memperhatikan hal yang ada dengan jeli dan teliti.

Demikian sejumlah ulasan mengenai tata cara setting DNS ubuntu server dengan tulus langkah mudah. Cara tersebut penting diketahui terlebih bagi orang yang ingin membutuhkannya untuk keperluan tertentu. DNS merupakan sistem yang bisa menerjemahkan nama domain menjadi IP address. Contohnya seperti gambar di atas, saya bisa mengetahui alamat IP dari facebook.com menggunakan tools nslookuo, dimana tools ini sendiri merupakan tools diagnotics untuk DNS.

Tanpa DNS, komputer tidak akan tau berapa alamat yang digunakan oleh facebook dan website lainnya juga. Dan jika alamat IP tidak diketahui, maka tidak akan terjadi koneksi antara mereka.

40. kapan dns di perkenalkan


DNS (Domain Name System) diperkenalkan pada tahun 1984 untuk menyeragamkan alamat di jaringan komputer yang ada.

Komputer yang tersambung dengan jaringan yang ada sudah melebihi 1000 komputer.
