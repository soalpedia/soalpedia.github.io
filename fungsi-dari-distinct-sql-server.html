Perbedaan utama nya ada pada sistem internal mereka masing - masing ,

MySQL misalnya menggunakan model multi engine, artinya penerapan secara teknis terkait penyimpanan data, pengambilan data dll diterapkan tergantung dari mekanisme dari engine yang digunakan (contoh MyISAM , ARIA, INNODB). untuk database lainnya biasanya single engine

perbedaan lainnya juga pada kemampuan fitur - fitur internal seperti

support terhadap clustered atau non clustered (heap) table, dan default model yang dianut.

misalkan saja MySQL dengan MyISAM hanya menganut heap table, namun Innodb menggunakan clustered , PostgreSQL AFAIK hanya menggunakan Heap, SQLServer dan Oracle dapat memilih menggunakan Clustered atau Heap, namun SQLServer menggunakan Clustered secara default, Oracle menggunakan Heap secara default (Untuk opsi Clustered pada oracle biasanya disebut sebagai IOT (Index Organized Table)). perbedaan ini mengakibatkan efek pada banyak hal , terutama performa secara umum.

fitur lainnya misalnya kemampuan vendor2 tersebut dalam melakukan Join, misalnya saja SQLServer dan Oracle (dan mungkin postgreSQL) mampu melakukan Algoritma Join tidak hanya nested join, namun MySQL sampai tulisan ini ditulis AFAIK, hanya dapat melakukan join secara nested, itu lah mengapa MySQL sangat tidak cocok digunakan untuk melakukan reporting, dan ini juga yang selalu dijadikan contoh oleh vendor - vendor “NoSQL” dengan jargonnya “Join is bad” . kemampuan join ini membuat perbedaan signifikan pada performa query yang melibatkan banyak tabel.

fitur lainnya lagi misalnya dalam pengadopsian model penyelesaian konkurensi, biasanya di dalam dunia RDBMS terdapat dua model secara umum yaitu Pesimistic Locking dan Optimistic Locking, setiap vendor memiliki pendekatan masing - masing yang memungkinkan juga memadukan beberapa model tersebut.

fitur lainnya lagi seperti penggunaan sistem Logging misalnya WAL (Write Ahead Log), untuk permasalahan durabilitas, dan juga fitur seperti replikasi , backup dan restore yang bisa jadi sangat berbeda.

11. fungsi perintah SQL dan penjelasannya


Ini adalah Beberapa Fungsinya Perintah SQL
1. Query
Perintah yang digunakan untuk menampilkan data dari database,. perintah yang digunakan adalah SELECT.

2. DML (Data Manipulation Language)
Perintah yang digunakan untuk manipulasi data seperti menambah, mengedit, menghapus data. Perintah yang digunakan INSERT, DELETE, UPDATE, MERGE

3. DDL (Data Definition Language)
Perintah yang digunakan untuk mendefiniskan struktur Objek database seperti membuat tabel, mengubah struktur tabel, menghapus tabel. Perintah yang digunakan CREATE, ALTER, DROP, TRUNCATE, RENAME

4. DCL (Data Control Language)
Perintah yang digunakan untuk mengatur perijinan akses terhadap database. Perintah yang digunakan GRANT, REVOKE

5. TCL (Transaction Control Language)
Perintah yang digunakan untuk menentukan suatu transaksi diterapkan secara permanen atau dibatalkan. Perintah yang digunakan COMMIT, ROLLBACK, SAVEPOINT.

21. apa fungsi SQL dalam pengembangan perangkat lunak​


SQL (Structured Query Language) adalah bahasa yang digunakan untuk mengelola dan mengakses basis data dalam pengembangan perangkat lunak. Fungsi SQL dalam pengembangan perangkat lunak sangat penting dan mencakup beberapa aspek utama, antara lain:

1. Manipulasi Data: SQL digunakan untuk mengambil, memasukkan, memperbarui, dan menghapus data dari basis data. Dengan SQL, pengembang dapat membuat, membaca, memperbarui, dan menghapus catatan data dalam aplikasi mereka.

2. Membangun dan Mengelola Basis Data: SQL memungkinkan pengembang untuk membuat basis data, tabel, indeks, dan relasi antara tabel. Ini adalah langkah penting dalam desain dan pengelolaan data dalam aplikasi.

3. Kueri Data: SQL memungkinkan pengembang untuk mengekstrak data yang diperlukan dari basis data dengan menggunakan pernyataan kueri (query). Ini penting untuk mengambil informasi yang dibutuhkan dari basis data yang besar.

4. Keamanan Data: SQL menyediakan mekanisme untuk mengontrol akses ke data. Ini termasuk pengaturan izin (permissions) untuk pengguna dan peran tertentu, sehingga hanya pengguna yang diizinkan yang dapat mengakses dan memanipulasi data.

5. Pengelolaan Transaksi: SQL mendukung transaksi, yang memungkinkan pengembang untuk mengelola operasi-operasi yang melibatkan perubahan data dalam basis data. Ini memastikan konsistensi data dan menghindari masalah seperti kehilangan data.

6. Penyimpanan Prosedur dan Fungsi: SQL memungkinkan pengembang untuk membuat penyimpanan prosedur (stored procedures) dan fungsi (functions) yang dapat digunakan dalam aplikasi. Hal ini membantu dalam pengelolaan logika bisnis di dalam basis data.

7. Integrasi dengan Aplikasi: SQL dapat diintegrasikan dengan banyak bahasa pemrograman dan kerangka kerja pengembangan perangkat lunak. Ini memungkinkan aplikasi untuk berinteraksi dengan basis data menggunakan SQL dalam kode mereka.

Dengan fungsi-fungsi ini, SQL menjadi komponen penting dalam pengembangan perangkat lunak yang berbasis data, memungkinkan pengembang untuk mengelola data dengan efisien dan aman.

25. 2. Jelaskan cara installasi SQL Server, apa yg perlu diperhatikan pada saat installasi tersebut, apa nama user sql yg pertama kali dibuat pada saat installasi.


Jawaban:

Instalasi SQL Server melibatkan beberapa langkah penting dan perhatian khusus. Berikut adalah langkah-langkah umum untuk menginstal SQL Server:

1. **Persiapan Sebelum Instalasi**:

- Pastikan sistem memenuhi persyaratan minimum sistem yang diperlukan oleh SQL Server.

- Backup data penting sebelum memulai proses instalasi.

2. **Jalankan Program Instalasi**:

- Jalankan file installer SQL Server.

3. **Pilih Jenis Instalasi**:

- SQL Server menyediakan beberapa opsi instalasi, seperti instalasi Default (standar) atau Custom (kustom). Pilih sesuai dengan kebutuhan Anda.

4. **Pilih Komponen Instalasi**:

- Tentukan komponen SQL Server yang ingin Anda instal. Misalnya, Database Engine, SQL Server Management Studio (SSMS), Integration Services, dll.

5. **Konfigurasi Instansi**:

- Jika Anda memilih instalasi Default, maka SQL Server akan membuat instansi dengan konfigurasi default. Jika Anda memilih Custom, Anda dapat menentukan nama instansi, port, dll.

6. **Pengaturan Server**:

- Tentukan apakah SQL Server akan berjalan dengan akun layanan lokal atau akun layanan jaringan. Perhatikan bahwa ini mempengaruhi hak akses SQL Server.

7. **Database Engine Configuration**:

- Konfigurasikan instance SQL Server, pengaturan autentikasi (Windows atau Mixed Mode), dan tentukan kata sandi untuk akun sa (sistem administrator).

8. **Instalasi dan Proses Pembaruan**:

- SQL Server akan melakukan instalasi dan mungkin memerlukan pembaruan atau paket tambahan.

9. **Penyelesaian Instalasi**:

- Setelah proses instalasi selesai, Anda akan menerima pemberitahuan.

Penting untuk diperhatikan:

- **Nama Pengguna SQL pertama**: Saat instalasi, Anda dapat membuat pengguna SQL pertama. Ini biasanya disebut "sa" (super administrator).

- **Kata sandi kuat**: Pastikan untuk menggunakan kata sandi yang kuat untuk akun "sa" dan catat dengan baik.

- **Firewall dan Izin Akses**: Pastikan izin firewall dan izin akses diperbarui untuk memungkinkan koneksi ke server SQL dari aplikasi atau jaringan yang membutuhkan akses.

Setelah instalasi, Anda dapat menggunakan SQL Server Management Studio (SSMS) untuk mengelola basis data dan konfigurasi server.


26. SQL Server is a structured database while MongoDB is an example of an


Jawaban:

unstructured database (data tak terstruktur)

Penjelasan:

SQL Server adalah sistem manajemen basis data relasional (RDBMS) yang menggunakan bahasa kueri terstruktur (SQL) untuk mengelola dan mengorganisir data secara terstruktur. SQL Server menggunakan tabel untuk menyimpan data dengan skema yang telah ditentukan sebelumnya, dan hubungan antara tabel ditetapkan dengan menggunakan kunci asing. SQL Server banyak digunakan dalam aplikasi bisnis dan sering menjadi pilihan utama untuk sistem transaksional yang membutuhkan konsistensi yang kuat dan kepatuhan ACID.

Di sisi lain, MongoDB adalah basis data NoSQL yang menggunakan model berorientasi dokumen untuk menyimpan data. MongoDB dianggap sebagai basis data tak terstruktur karena tidak memerlukan skema atau struktur tabel yang telah ditentukan sebelumnya. Sebaliknya, data disimpan sebagai dokumen dalam sebuah koleksi, dan setiap dokumen dapat memiliki struktur atau skema yang berbeda. MongoDB sangat scalable, fleksibel, dan cepat, sehingga menjadi pilihan populer untuk aplikasi web dan mobile yang membutuhkan throughput yang tinggi dan laten rendah.

29. Fungsi dari Php fungsi dari sql


PHP dapat disebut dengan bahasa pemrograman umum yang berarti php dapat disematkan ke dalam kode HTML, atau dapat digunakan dalam kombinasi dengan berbagai sistem templat web, sistem manajemen konten web, dan kerangka kerja web.  

Sedangkan SQL sendiri merupakan singkatan dari Structured Query Language. Secara umum SQL dapat diartikan sebagai bahasa yang digunakan dalam mengakses data.  

Pembahasan

Fungsi dari PHP yakni membuat atau mengembangkan situs web statis atau situs web dinamis atau aplikasi Web. PHP digunakan karena untuk membuat website dinamis bisa dipakai untuk menyimpan data ke dalam database, membuat halaman yang dapat berubah-ubah sesuai dengan input  user, memproses form, dan yang lain-lain. Dalam membuat file PHP dapat digabungkan dengan menggunakan tag html, kemudian ketika tanpa menggunakan tag html apa pun disebut sebagai file PHP Murni. Server menginterpretasikan kode PHP lalu mengeluarkan hasilnya sebagai kode HTML ke browser web. 36. Tugas :Cari pengertiana. Microsoft SQL Serverb. MySQLC. Oracle​


a. Microsoft SQL Server

Microsoft SQL Server adalah sebuah sistem manajemen basis data relasional (RDBMS) produk Microsoft. Bahasa kueri utamanya adalah Transact-SQL yang merupakan implementasi dari SQL standar ANSI/ISO yang digunakan oleh Microsoft dan Sybase.

b. MySQL

MySQL adalah salah satu jenis database yang bersifat open source. Website dan aplikasi berbasis mobile memerlukan database server untuk menampung informasi yang banyak.

c. Oracle

Basis data Oracle adalah basis data relasional yang terdiri dari kumpulan data dalam suatu sistem manajemen basis data RDBMS. Perusahaan perangkat lunak Oracle memasarkan jenis basis data ini untuk bermacam-macam aplikasi yang bisa berjalan pada banyak jenis dan merk perangkat keras komputer (platform).
