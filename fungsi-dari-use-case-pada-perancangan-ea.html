3. Hubungan dimana satu use case memperluas perilaku dari use case yang lain merupakan pengertian dari salah satu relasi dalam use case yakni


Jawaban:

"Extend" relasi

Penjelasan:

Dalam analisis use case, relasi "extend" digunakan untuk menggambarkan hubungan antara dua use case di mana salah satu use case (yang disebut use case yang meng-extend) memperluas perilaku use case lainnya (yang disebut use case yang di-extend). Use case yang meng-extend menambahkan langkah-langkah atau fungsionalitas tambahan ke dalam use case yang di-extend.

Dalam relasi "extend", use case yang di-extend adalah use case yang lebih umum dan mencakup perilaku dasar, sedangkan use case yang meng-extend adalah use case yang spesifik dan hanya diterapkan dalam kondisi tertentu atau sebagai perluasan opsional.

Contoh penggunaan relasi "extend" adalah dalam sistem pemesanan online. Use case utama adalah "Pesan Produk", yang mencakup langkah-langkah untuk memilih produk, menambahkannya ke keranjang belanja, dan menyelesaikan pembayaran. Namun, ada juga use case opsional yang meng-extend use case utama, seperti "Tambahkan Kupon Diskon" atau "Tambahkan Produk Tambahan". Use case-use case ini hanya diterapkan jika pengguna ingin menggunakan kupon diskon atau menambahkan produk tambahan ke pesanan mereka, tetapi tidak wajib dilakukan.

Dengan menggunakan relasi "extend", analisis use case dapat menunjukkan bagaimana use case-use case spesifik dapat memperluas atau melengkapi perilaku use case yang lebih umum, memberikan fleksibilitas dalam sistem untuk menangani situasi khusus tanpa mengubah use case yang di-extend.

4. pengertian extend dan include dalam use case


Extend (Perluasan): Extend digunakan ketika suatu use case memiliki aksi tambahan yang dapat terjadi tergantung pada kondisi tertentu. Use case yang diperluas adalah use case utama, sedangkan use case yang memperluas adalah ekstensi. Use case ekstensi hanya terjadi jika kondisi tertentu dipenuhi. Dengan kata lain, use case ekstensi "meng-extend" fungsionalitas use case utama.

Include (Penggabungan): Include digunakan ketika suatu use case membutuhkan fungsionalitas dari use case lain untuk menyelesaikan tugasnya. Use case yang membutuhkan fungsionalitas tambahan adalah use case inklusi, sedangkan use case yang menyediakan fungsionalitas tambahan adalah use case yang diikutsertakan. Use case inklusi selalu terjadi dan dipanggil oleh use case utama.

Pembahasan

Contoh penggunaan extend:

Misalkan kita memiliki use case "Pesan Tiket" sebagai use case utama. Namun, terdapat ekstensi yang disebut "Tawarkan Upgrade Kelas" yang hanya terjadi jika pengguna memilih opsi upgrade kelas saat memesan tiket. Use case "Tawarkan Upgrade Kelas" adalah contoh dari extend yang memperluas fungsionalitas use case utama "Pesan Tiket".

Contoh penggunaan include:

Misalkan kita memiliki use case "Bayar Tagihan" sebagai use case utama. Di dalam use case ini, terdapat fungsionalitas untuk "Verifikasi Pembayaran". Namun, "Verifikasi Pembayaran" juga dapat digunakan oleh use case lain, seperti "Lihat Riwayat Pembayaran". Dalam hal ini, use case "Bayar Tagihan" akan mencakup atau mengikutsertakan use case "Verifikasi Pembayaran".

Penjelasan:

1. Actor

Meskipun banyak yang menyangka bahwa aktor adalah sebuah diagram, tetapi nyatanya tidak. Aktor bukanlah bagian dari diagram. Aktor di sini mempunyai peran begitu penting, yaitu menciptakan use case jadi lebih mudah. Aktor menjelaskan siapa yang berinteraksi dengan sistem. Selain itu aktor juga akan memberikan informasi sekaligus menerima informasi dari sistem. Hebatnya, kedua aktivitas tersebut bisa terjadi secara bersamaan, Sob. Namun, aktor sama sekali tidak mengontrol sistem, melainkan hanya memberi gambaran tentang hubungannya dengan sistem tersebut.

2. Use case

Komponen selanjutnya adalah use case. Use case merupakan komponen gambaran fungsional dalam sebuah sistem. Dengan begitu, pengguna atau konsumen dapat mengetahui setiap fungsi yang dibangun dalam sistem tersebut.

3. Association

Association adalah teknik yang digunakan untuk mengidentifikasi interaksi yang dilakukan oleh aktor tertentu dengan use case tertentu. Ini digambarkan dengan garis penghubung antara aktor dengan use case.

4. Generalisation

Generalisasi adalah hubungan antara dua use case atau dua aktor. Dimana salah satunya meng-inherit dan menambahkan atau override sifat dari yang lainnya.

5. Dependency

Dependency adalah relasi yang terbagi menjadi dua jenis, yaitu include dan exclude. Include berfungsi untuk mengidentifikasi hubungan atau relasi antara dua use case, dimana use case yang satu akan memanggil use case lainnya. Sementara exclude merupakan jenis yang jika dilakukan pemanggilan memerlukan suatu kondisi tertentu dan akan terjadi dependensi.

23. sebutkan jenis jenis use case​


Jawaban:

Use case utama (Main use case): Use case utama adalah skenario utama yang menjelaskan interaksi antara pengguna dan sistem. Use case ini mewakili tindakan atau proses utama yang ingin dicapai oleh pengguna dengan menggunakan sistem.

Use case alternatif (Alternative use case): Use case alternatif menjelaskan skenario tambahan atau alternatif yang terjadi saat pengguna menggunakan sistem. Use case alternatif diaktifkan ketika kondisi tertentu terpenuhi atau ketika terjadi kesalahan dalam use case utama.

Use case sekunder (Secondary use case): Use case sekunder adalah use case yang mendukung use case utama dan membantu pengguna dalam mencapai tujuannya. Use case ini mungkin tidak perlu dilakukan dalam setiap interaksi dengan sistem, namun dapat diaktifkan ketika diperlukan.

Use case eksternal (External use case): Use case eksternal mewakili interaksi antara sistem dan pihak eksternal, seperti sistem atau aplikasi lain. Contoh use case eksternal adalah integrasi sistem dengan sistem pihak ketiga atau dengan sistem perbankan.

Use case bisnis (Business use case): Use case bisnis mewakili proses bisnis yang ingin dicapai oleh pengguna atau organisasi. Use case ini membantu mengidentifikasi kebutuhan bisnis dan memastikan bahwa sistem memenuhi kebutuhan tersebut.

Use case teknis (Technical use case): Use case teknis mewakili interaksi teknis antara sistem dan infrastruktur teknologi, seperti jaringan atau database. Use case ini membantu memastikan bahwa sistem berfungsi dengan benar dalam lingkungan teknis yang sesuai.

Use case pengujian (Testing use case): Use case pengujian mewakili skenario pengujian yang digunakan untuk menguji sistem dan memastikan bahwa sistem berfungsi sesuai dengan kebutuhan dan spesifikasi.
